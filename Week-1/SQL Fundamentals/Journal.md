# 📘 Learning Journal – SQL Assignment (ClassicModels)

## 🧠 What I have understood

Through this assignment, I have gained a clearer understanding of how SQL queries are structured and optimized for real-world database operations.

### 1. Table Chaining and Projection
I understood how tables can be connected using **JOIN operations** to combine related data from multiple tables. I also learned how to use **projection operations (SELECT clause)** to extract only the required columns for the final output.

---

### 2. Query Optimization Techniques
I learned how to write more efficient queries by understanding when to use different SQL operations such as:
- `INNER JOIN` for matching related records between tables
- `DISTINCT` to remove duplicate values from results
- Proper JOIN selection to improve readability and performance

This helped me understand how query structure affects efficiency.

---

### 3. Difference between WITH clause and VIEW
I understood the key difference between:
- **WITH clause (CTE):** Used for temporary result sets that are not stored in the database.
- **VIEW:** A virtual table that is stored in the database and can be reused like a real table.

This clarified how SQL manages temporary vs reusable query structures.

---

### 4. Transaction Control Language (TCL)
I understood the concept of **TCL commands**, especially:
- `COMMIT` → to permanently save changes
- `ROLLBACK` → to undo changes in case of errors
- Transactions → to group multiple SQL operations into a single logical unit

This helps maintain data integrity and control database changes safely.

Overall, this assignment helped me improve my understanding of relational database concepts, query optimization, and transaction management in SQL.
